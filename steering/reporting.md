# 보고서 작성 규칙

## 장애 복구 보고서
장애 복구 시 반드시 아래 형식의 markdown 보고서를 작성한다.

```markdown
# 장애 보고서

## 개요
- 발생일시:
- 복구일시:
- 영향 범위:
- 심각도: Critical / High / Medium / Low

## 장애 원인
- 근본 원인(Root Cause) 상세 기술

## 타임라인
| 시각 | 내용 |
|------|------|
| HH:MM | 장애 감지 |
| HH:MM | 원인 분석 |
| HH:MM | 조치 수행 |
| HH:MM | 복구 완료 |

## 조치 사항
- 수행한 조치 내용 상세 기술

## 재발 방지 대책
- 향후 동일 장애를 방지하기 위한 개선 방안
```

## AWS 서비스 변경 보고서
AWS 서비스 설정 변경 시 반드시 아래 형식의 markdown 보고서를 작성한다.

```markdown
# AWS 서비스 변경 보고서

## 개요
- 변경일시:
- 변경 대상 서비스:
- 변경 목적:

## 변경 전 설정
- 기존 설정값 상세 기술

## 변경 후 설정
- 변경된 설정값 상세 기술

## 변경 사유
- 왜 이 변경이 필요한지 상세 기술

## 영향도 분석
- 이 변경으로 인해 영향을 받는 서비스/리소스

## 롤백 계획
- 문제 발생 시 원복 방법
```
