# 코드 작성 규칙

## 주석
- 코드 작성 시 핵심 로직, 비즈니스 규칙, 복잡한 알고리즘에는 반드시 한국어 주석을 작성한다.
- 주석은 "왜(Why)" 이 코드가 필요한지를 설명하는 데 집중한다.
- 함수/클래스 단위에는 목적과 주요 파라미터를 한국어로 설명한다.

## 코드 리뷰
- 코드 작성이 완료되면 반드시 코드 리뷰를 실시한다.
- 리뷰 항목: 보안 취약점, 성능 이슈, 코드 가독성, 에러 처리, 중복 코드
- 리뷰 결과에서 발견된 문제는 즉시 수정한 후 최종 코드를 확정한다.

## Git 커밋
- 코드 작성 및 리뷰 완료 후 반드시 `git commit`을 실행한다.
- 커밋 메시지는 한국어로 작성하며, 변경 사항의 목적을 명확히 기술한다.
- 형식: `[카테고리] 변경 내용 요약` (예: `[기능추가] 사용자 인증 API 구현`)

## 비용 절감 / 효율화
- 모든 작업 수행 시 비용 절감과 효율화 방안을 항상 고려한다.
- AWS 리소스 사용 시 불필요한 리소스 생성을 지양하고, 적절한 인스턴스 타입/서비스 티어를 선택한다.
- 코드 레벨에서도 불필요한 API 호출, 중복 연산, 비효율적 데이터 처리를 줄인다.
